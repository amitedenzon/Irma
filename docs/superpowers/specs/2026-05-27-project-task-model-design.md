# Project + Task Model — Manual-First PMO Slice

**Status:** Draft for review
**Date:** 2026-05-27
**Owner:** amitedenzon
**Supersedes:** the auto-observed `StandupBrief` shape introduced in Phase 3

---

## 1. Context

Irma's Phases 1–3 shipped a passive observation pipeline: `TimeAgent` (Google Calendar) and `CodebaseAgent` (local git) emit `Signal`s, `LeadAgent` synthesizes them into a `StandupBrief`, and the companion sprite reacts via SSE. That model assumes two things that do not hold for this user:

1. **Code lives locally** — in practice, most of the user's code is on an SSH server. `CodebaseAgent` is largely blind.
2. **Work lives on the calendar** — in practice, only collaborative items (e.g. meetings with Prof. Gal) land on the calendar. Lectures, homework, and individual research time do not.

The consequence: a brief synthesized from auto-observed signals describes a small slice of reality. The user's actual workflow is task-driven, with goals at multiple cadences (daily / weekly / monthly).

This spec defines the first slice of the app that makes Irma a real PMO assistant rather than an observer: **a manual Project + Task model with horizon-aware synthesis.**

## 2. Goals & non-goals

**Goals**
- First-class `Project` and `Task` entities, manually managed by the user.
- Four brief horizons (`day`, `week`, `month`, `all`) generated by a single horizon-aware synthesizer.
- Minimal dashboard that supports the full CRUD loop and renders each brief.
- Calendar signals attribute to projects automatically via keyword match; no new auto-discovery heuristics.

**Non-goals (this slice)**
- Local LLM inference (gpt-oss-120B on Mac GPU) — deferred subsystem.
- Outbound channels (email, native notifications) — deferred subsystem.
- Scheduled digests (auto-delivered daily/weekly/monthly emails) — deferred subsystem.
- Reminder engine, calendar write-ops, SSH-aware codebase observer — deferred.
- Drag-and-drop, kanban, calendar grid, undo, keyboard shortcuts beyond Enter/Cmd+Enter.

## 3. Decisions log

Choices made during brainstorming, with rationale, so future edits know what is load-bearing vs. incidental.

| # | Decision | Rationale |
|---|----------|-----------|
| D1 | First subsystem to design: **Project model** (over local LLM, outbound channels, digests). | Smallest pure-backend change with the largest unlock; every other subsystem operates on "projects" rather than "git repos." |
| D2 | Project lifecycle: **manual only**, no discovery heuristic. | Predictable, simple, and the user explicitly preferred control over grouping (e.g. monorepos, multi-repo projects). |
| D3 | Project schema: **minimal core + goals/deadlines** (`goals[]` free text + `target_date`). | The user explicitly framed work in terms of "daily/weekly/monthly goals." Skipping this would require a redesign later. |
| D4 | Goal cadence: **free-text goals on Project**, no first-class `Goal` entity. Horizon views are date-window queries the LLM reasons over. | Avoids a CRUD surface no one will maintain ("set new weekly goals every Monday"); maximizes LLM leverage. |
| D5 | Signal attribution: **at write time, in the store**, via keyword match (calendar only). | Observers stay dumb. One place to evolve match logic. Enables `/signals?project_id=X` queries. |
| D6 | Reframe (after D1–D5): **Task is the primary unit**; signals are secondary context. | The user's actual workflow is task-driven, not observation-driven (SSH code, off-calendar work). |
| D7 | Coexistence with old `/standup`: **replace** with `/brief/*`; delete the old endpoint and `StandupBrief` shape. | The old brief was Phase 3 placeholder; preserving it would be vestigial. Clean break, single Pydantic shape. |
| D8 | Brief surface: **four named endpoints** (`/today`, `/week`, `/month`, `/overview`), one synthesizer, per-horizon cache. | URLs are self-documenting; per-horizon cache means editing a task today doesn't re-synth the month view. |
| D9 | UI scope: **backend + minimal dashboard** (project list, per-project tasks, horizon-switchable brief). | Enough to actually use day-to-day, nothing fancy. |
| D10 | `CodebaseAgent`: **disabled behind a flag**, code retained. | Deletion is rework if an SSH-aware variant comes back later. Flag costs nothing. |

## 4. Data model

Two new entities, both in the existing SQLite store next to `signals`. UUIDv4 string IDs, ISO timestamps.

```python
# services/api/src/irma_api/models/project.py
class ProjectStatus(StrEnum):
    active = "active"
    paused = "paused"
    archived = "archived"

class Project(BaseModel):
    id: str
    name: str                            # required, unique case-insensitive, 1–80 chars
    description: str = ""
    status: ProjectStatus = active
    priority: int = 2                    # 1 = high, 2 = med, 3 = low
    calendar_keywords: list[str] = []    # lowercased, deduped, each ≥ 2 chars
    goals: list[str] = []                # free-text bullets
    target_date: date | None = None
    created_at: datetime
    updated_at: datetime
```

```python
# services/api/src/irma_api/models/task.py
class TaskStatus(StrEnum):
    todo = "todo"
    doing = "doing"
    done = "done"
    blocked = "blocked"

class Task(BaseModel):
    id: str
    project_id: str                      # FK → project.id, ON DELETE RESTRICT
    title: str                           # 1–200 chars
    notes: str = ""
    status: TaskStatus = todo
    due_date: date | None = None         # hard deadline; overdue is allowed
    scheduled_for: date | None = None    # the day the user intends to work on it
    estimated_minutes: int | None = None # > 0 if set
    created_at: datetime
    updated_at: datetime
    completed_at: datetime | None = None # auto-stamped on status→done; cleared on unset
```

### 4.1 Store changes

- New migration step `0002_projects_tasks.sql` (idempotent, additive):
  - Tables `project`, `task`, `brief_cache` (see §6.3 for the cache schema).
  - Indexes: `task(project_id)`, `task(scheduled_for)`, `task(due_date, status)`.
  - Add nullable column `signals.project_id TEXT NULL` + index on it. Existing rows untouched.
- New DAOs `ProjectRepo`, `TaskRepo` in `store/repos/`. Pure data access; no business logic.
- New `store/errors.py`: `NotFoundError`, `ConflictError`.

### 4.2 Signal attribution

```python
# store/sqlite.py
async def add(self, sig: Signal) -> None:
    project_id = await self._match_project(sig) if sig.source == "calendar" else None
    await self._insert(sig, project_id)

async def _match_project(self, sig: Signal) -> str | None:
    haystack = f"{sig.title} {sig.detail}".lower()
    for p in await self.projects.list(status="active"):  # sorted: priority, name
        if any(kw in haystack for kw in p.calendar_keywords):
            return p.id
    return None
```

Deterministic ordering by `(priority, name)` resolves keyword overlaps; first match wins. Logged at `debug`.

### 4.3 What we are not adding

- No `Goal` entity (D4).
- No tags, labels, subtasks.
- No soft delete on `Task`. Soft delete on `Project` via `status = archived`.

## 5. API surface

All routes under `/api/v1`. JSON in/out, Pydantic v2 schemas, async handlers. Error responses uniform: `{"error": "<machine_code>", "detail": "<human msg>"}`.

### 5.1 Projects

| Method | Path | Body / Query | Returns |
|---|---|---|---|
| `GET` | `/projects` | `?status=` (repeatable; default `active`) | `list[Project]`, sorted by `priority` then `name` |
| `POST` | `/projects` | `ProjectCreate` | `Project`, `201`. `409` on duplicate name |
| `GET` | `/projects/{id}` | — | `Project`. `404` if missing |
| `PATCH` | `/projects/{id}` | `ProjectUpdate` (all fields optional) | `Project`. `404` / `409` |
| `DELETE` | `/projects/{id}` | — | `204`. `409` if any non-`done` task remains (archive instead) |

### 5.2 Tasks

| Method | Path | Body / Query | Returns |
|---|---|---|---|
| `GET` | `/tasks` | `?project_id=`, `?status=` (repeatable), `?scheduled_from=&scheduled_to=`, `?due_before=` | `list[Task]`, sorted by `due_date NULLS LAST, scheduled_for NULLS LAST, created_at` |
| `POST` | `/tasks` | `TaskCreate` (project_id + title required) | `Task`, `201`. `404` if project missing, `409` if project archived |
| `GET` | `/tasks/{id}` | — | `Task` |
| `PATCH` | `/tasks/{id}` | `TaskUpdate` — partial. Setting `status=done` auto-stamps `completed_at`; unsetting clears it | `Task` |
| `DELETE` | `/tasks/{id}` | — | `204` |
| `POST` | `/tasks/{id}/complete` | — | shortcut for `PATCH {status: "done"}`; idempotent |

### 5.3 Brief

| Method | Path | Horizon | Content |
|---|---|---|---|
| `GET` | `/brief/today` | `day` | Tasks `scheduled_for=today` + `due_date ≤ today+1` + today's calendar items. Recommended focus + conflicts. |
| `GET` | `/brief/week` | `week` | Open tasks in the current ISO week window. Per-project trajectory. Week calendar overview. Off-track callouts. |
| `GET` | `/brief/month` | `month` | Project-level rollup against `target_date` and `goals[]`. Cross-project conflicts. Strategic recommendation. |
| `GET` | `/brief/overview` | `all` | Time-window-free snapshot across all `active` projects: status, goals, recent momentum. |
| `POST` | `/refresh` | — | Existing endpoint; also invalidates all four brief cache rows. |
| `GET` | `/stream` | — | Existing SSE stream. Unchanged. |

### 5.4 Validation rules (enforced in Pydantic)

- `Project.name`: 1–80 chars, trimmed, unique case-insensitive.
- `Project.priority`: int in `{1, 2, 3}`.
- `Project.calendar_keywords`: each ≥ 2 chars, lowercased + deduped on assignment.
- `Task.title`: 1–200 chars.
- `Task.estimated_minutes`: `> 0` if set.
- `Task.due_date >= today` is **not** enforced — overdue tasks are real.

### 5.5 Deprecated / removed

- `GET /standup` removed (no external clients).
- `models.brief.StandupBrief` deleted.
- `routers/standup.py` deleted.

## 6. Synthesis flow

### 6.1 `LeadAgent` (rewritten)

```python
Horizon = Literal["day", "week", "month", "all"]

class LeadAgent:
    def __init__(self, anthropic, projects, tasks, signals, state_bus): ...

    async def synthesize(self, horizon: Horizon) -> Brief:
        async with self.state_bus.transition_to("thinking"):
            ctx = await self._build_context(horizon)
            messages = self._compose_prompt(horizon, ctx)
            raw = await self._call_claude(messages)
            return self._parse_brief(horizon, raw)
```

- `_build_context(horizon)` runs three reads in parallel via `asyncio.gather`: active projects + tasks in the horizon's date window + recent calendar signals in that window. Returns a typed `SynthesisContext`.
- `_compose_prompt(horizon, ctx)` builds the Anthropic Messages payload. System prompt = Irma persona, loaded once at startup from `agents/prompts/irma_persona.md`. User message = horizon-specific template populated with serialized projects/tasks/signals.
- `_call_claude` uses the existing async `anthropic` client. Model from `ANTHROPIC_MODEL`. Wrapped in `tenacity` retry (3 attempts, exp backoff, 429/5xx triggers).
- `_parse_brief` validates with `Brief.model_validate_json(raw)`. On failure, one retry with explicit schema-mismatch message; second failure → `502`.
- `transition_to("thinking")` context manager flips `AgentState`. On exit: `alert` if `Brief.conflicts` non-empty, else `idle`.

### 6.2 `Brief` shape

```python
class Brief(BaseModel):
    horizon: Literal["day", "week", "month", "all"]
    generated_at: datetime
    focus: list[FocusItem]                  # tasks/events to act on
    project_status: list[ProjectStatusItem] # per-project rollup
    conflicts: list[str]
    recommendation: str                     # 1–3 sentences, Irma voice
    narrative: str                          # prose
```

Sections may be empty depending on horizon; the dashboard hides empty sections.

### 6.3 Caching

- New table `brief_cache(horizon PRIMARY KEY, payload_json, computed_at, inputs_hash)`.
- `inputs_hash` = `sha256(sorted task ids + their updated_at + relevant signal hashes for that horizon's window + active project ids + their updated_at)`.
- On `GET /brief/<horizon>`: recompute `inputs_hash`; if it matches the cached row, return cached. Else re-synth, write back.
- `POST /refresh` deletes all four rows.

### 6.4 Observer changes

- **`TimeAgent`** — unchanged behavior. The store-layer keyword match (4.2) attributes its emissions to projects.
- **`CodebaseAgent`** — registration removed; gated on `CODEBASE_AGENT_ENABLED=false` (default off). Module and tests retained. Docstring annotated: *"Disabled pending SSH-aware variant."*
- **Scheduler** — keep the 15-minute `TimeAgent.collect()` job. Drop the periodic brief precompute job; briefs re-synth lazily on cache miss.

- Landing screen. Sub-tabs: `Today` · `Week` · `Month` · `Overview`, each calls `GET /brief/<horizon>`.
- Renders the unified `Brief`. Empty sections do not render.
- Checkbox on a `FocusItem` of kind `task` calls `POST /tasks/{id}/complete` and optimistically strikes through.

### 7.2 ProjectsView

- Two-pane: project list (left) + project detail (right).
- Project list sorted by `priority` then `name`; archived collapsed by default.
- Project detail header shows name, status, priority, target date (days remaining), goals, action buttons (`edit`, `archive`).
- Task list under the header: inline expand-on-click to edit `notes`, `status`, `due_date`, `scheduled_for`, `estimated_minutes`. Saves on blur or Cmd+Enter.
- Inline `+ add task` row reveals a single-row form (title + optional dates); Enter saves.

## 8. Errors

| Layer | Failure | Response |
|---|---|---|
| Store | row not found | `NotFoundError` → router → `404 {"error": "not_found"}` |
| Store | unique / FK violation | `ConflictError` → router → `409 {"error": "conflict", "detail": "..."}` |
| Synthesis | Anthropic 429/5xx after retries | `503 {"error": "synthesis_unavailable"}`, state bus → `alert` |
| Synthesis | JSON parse failure × 2 | `502 {"error": "synthesis_parse_failed", "raw_excerpt": "..."}` (200-char excerpt, logged at `warning`) |
| Synthesis | empty context (no projects/tasks/signals) | Skip LLM call. Return stub `Brief(recommendation="Add a project to get started.", narrative="")` |
| Observer (`TimeAgent`) | OAuth missing / expired | Emit zero signals + log `WARNING`. State bus stays `idle`. No exception. |
| Observer (`TimeAgent`) | network failure | tenacity retries; final failure logged, zero signals that cycle |

## 11. Out of scope (explicitly deferred to future specs)

- **Local LLM serving** (gpt-oss-120B on Mac GPU): orthogonal infra, can swap in as alternate synthesizer later.
- **Outbound channels** (email via Gmail API, native macOS notifications): needed before "scheduled digests" makes sense.
- **Scheduled digests** (daily/weekly/monthly auto-delivered): depends on outbound channels.
- **Reminder engine** (time-aware nudges): depends on outbound channels.
- **Calendar write-ops** (create reminder events, block focus time, RSVP): broader OAuth scope, UX risk; defer.
- **SSH-aware `CodebaseAgent`**: useful when re-introduced, but the current `CodebaseAgent` is local-only and misses most of the user's code.
